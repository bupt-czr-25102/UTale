## 二、详细使用指南

### 1. 如何引用CSS变量

**在WXSS文件中使用**
```css
/* 示例1：设置按钮背景色 */
.my-button {
  background-color: var(--primary-color);
  color: white;
  /* 其他样式 */
}

/* 示例2：设置文本颜色 */
.my-text {
  color: var(--text-secondary);
  /* 其他样式 */
}
```

**在组件样式中使用**
```html
<!-- 在wxml中直接使用预定义类名 -->
<view class="card">
  <text class="text-lg text-primary">卡片标题</text>
  <text class="text-secondary">卡片内容</text>
  <button class="btn">确定</button>
</view>
```

### 2. 如何使用预定义类名

**按钮使用**
```html
<!-- 主按钮 -->
<button class="btn">提交</button>

<!-- 次要按钮 -->

```

**列表使用**
```html
<!-- 列表容器 -->
<view class="container">
  <!-- 列表项 -->
  <view class="item">
    <text class="text-primary">列表项标题</text>
    <text class="text-secondary">列表项内容</text>
  </view>
  <view class="item">
    <text class="text-primary">另一个列表项</text>
    <text class="text-secondary">另一个列表项内容</text>
  </view>

```

**布局使用**
```html

  <text>水平垂直居中的文本</text>

  <text>左侧文本</text>

```

## 三、注意事项

### 1. 颜色使用要求
- **必须保持主色调一致**：所有页面和组件应使用 `var(--primary-color)` 作为主色调，确保整个应用风格统一
- **其他颜色可自由选择**：次要颜色、背景色等可根据需要在各自的组件中调整

### 2. 字体使用要求
- **字体大小建议保持不变**：全局已定义的字体大小（如 .text-base、.text-lg 等）经过优化，建议直接使用，不要自行修改
- **字体样式可自定义**：字体粗细、行高等可在各自的组件中调整

### 3. 样式修改规则
- **禁止修改全局样式文件**：不要直接修改 `app.wxss` 文件，以免影响其他模块
- **在各自模块中调整**：如需自定义样式，应在各自模块的 `.wxss` 文件中添加，通过类名优先级覆盖全局样式

### 4. 命名规范
- **类名命名**：自定义类名建议使用语义化的命名，如 `.user-card`、`.task-item` 等
- **避免冲突**：自定义类名应避免与全局类名冲突，如已使用 `.btn` 作为全局按钮类名，自定义按钮可使用 `.custom-btn` 等

### 5. 性能优化
- **减少样式嵌套**：避免过深的样式嵌套，影响渲染性能
- **复用样式**：尽量使用全局预定义的类名，减少重复代码
- **按需加载**：只在需要的地方引入样式，避免不必要的样式加载

## 四、常见问题解答

### 1. 如何修改按钮的大小？
在各自模块的 `.wxss` 文件中添加自定义类：
```css
/* 小按钮 */
.btn-small {
  padding: 10rpx 20rpx;
  font-size: 24rpx;
}

/* 大按钮 */
.btn-large {
  padding: 25rpx 50rpx;
  font-size: 32rpx;
}
```

### 2. 如何添加新的颜色变量？
在各自模块的 `.wxss` 文件中添加：
```css
/* 模块内颜色变量 */
:root {
  --module-specific-color: #ff6b6b;
}

/* 使用 */
.module-element {
  color: var(--module-specific-color);
}
```

### 3. 如何覆盖全局样式？
通过增加类名优先级：
```css
/* 全局样式 */
.btn {
  background-color: var(--primary-color);
}

/* 模块内覆盖 */
.custom-btn.btn {
  background-color: #ff6b6b;
}
```

## 五、总结

全局样式文件为项目提供了统一的设计规范，包括颜色系统、字体设置、组件样式和布局工具类。所有组员应遵循以下原则：

1. **主色调必须一致**：使用 `var(--primary-color)` 作为主色调，确保应用风格统一
2. **字体大小建议不变**：直接使用预定义的字体大小类名
3. **禁止修改全局样式文件**：在各自模块中调整样式
4. **充分利用预定义类**：减少重复代码，提高开发效率

通过统一的样式规范，可以确保整个应用的视觉效果一致，同时为开发过程提供便利。如有任何问题，请及时与负责人沟通。
