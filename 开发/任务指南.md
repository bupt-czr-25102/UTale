# 雏雁项目任务指南

## 一、基础框架使用说明

### 1. 框架结构概述

我们已经为项目搭建了完整的基础框架，包括前端微信小程序和后端Flask服务。框架采用模块化设计，每个功能模块都有独立的目录结构，便于开发和整合。

```
雏雁项目/
├── frontend/              # 前端微信小程序代码
│   ├── pages/            # 页面文件目录
│   │   ├── diary/        # AI日记模块
│   │   ├── video/        # AI视频模块
│   │   ├── schedule/     # 日程管理模块
│   │   └── community/    # 社区模块
│   ├── utils/            # 通用工具函数
│   ├── app.js            # 小程序入口文件
│   ├── app.json          # 小程序配置文件
│   └── app.wxss          # 小程序全局样式
└── backend/              # 后端Flask代码
    ├── modules/          # 功能模块目录
    │   ├── diary/        # AI日记模块API
    │   ├── video/        # AI视频模块API
    │   ├── schedule/     # 日程管理模块API
    │   └── community/    # 社区模块API
    ├── app.py            # Flask应用入口
    ├── config.py         # 配置文件
    └── requirements.txt  # 依赖包列表
```

### 2. 如何开始开发

#### 前端开发（微信小程序）
1. 下载并安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 打开微信开发者工具，选择「导入项目」
3. 选择 `frontend` 目录作为项目目录
4. 在 `pages/[your-module]/` 目录下开发你的功能模块

#### 后端开发（Flask）
1. 安装Python 3.8+和pip
2. 打开命令行，进入 `backend` 目录
3. 安装依赖包：`pip install -r requirements.txt`
4. 在 `modules/[your-module]/routes.py` 文件中开发你的API接口
5. 启动Flask服务：`python app.py`
6. 访问 `http://localhost:5000/api` 检查服务是否正常运行

### 3. 框架使用提示

- **文件命名**：严格按照统一开发标准中的命名规范命名文件和变量
- **代码结构**：在自己负责的模块目录内开发，不要修改其他模块的代码
- **API调用**：前端通过 `app.js` 中的 `globalData.apiBaseUrl` 调用后端API
- **示例代码**：框架中已经包含了简单的示例代码，你可以参考这些代码进行开发

## 二、定期任务安排

为了确保项目进度，我们建立了以下定期任务：

### 1. 每周进度汇报
- **时间**：每周日晚上22:30前
- **内容**：
  - 本周完成的功能
  - 遇到的问题和解决方案（有可随时说）
  - 下周计划开发的功能
  - 需要团队协助的事项（有可随时说）
- **提交方式**：在微信群中发送文字汇报

### 2. 每周代码提交
- **时间**：每周日晚上10:00前
- **内容**：
  - 将本周开发的代码打包成ZIP文件
  - 文件名格式：`模块名_开发者姓名_日期.zip`（如：`diary_张三_20260126.zip`）
  - 包含前端和后端的所有修改文件
- **提交方式**：发送给项目负责人

### 3. 阶段性成果检查
- **时间**：项目启动后第7天、第14天
- **内容**：
  - 展示已完成的功能
  - 演示基本的用户流程
  - 讨论下一阶段的开发重点
- **形式**：留下视频、图片等资料（便于后期ppt制作）

## 四、统一开发标准说明

### 1. 标准用途
- 确保团队成员使用一致的技术栈
- 统一代码风格和命名规范
- 降低代码整合的难度
- 提高代码的可读性和可维护性

### 2. 重要注意事项

#### 技术栈选择
- 严格按照文档中指定的技术栈进行开发（微信小程序+Python Flask+MySQL+DeepSeek API）
- 不要随意更换技术框架，如有特殊需求请先与负责人沟通

#### 代码规范
- 遵循文档中的命名规范（变量命名、函数命名、文件命名）
- 保持代码的简洁和可读性
- 适当添加注释，说明代码的功能和逻辑

#### 数据库设计
- 按照文档中的规范设计数据库表结构
- 使用统一的字段命名和数据类型
- 注意数据的完整性和一致性

#### API设计
- 采用RESTful API设计风格
- 统一接口的请求和响应格式
- 确保API的安全性和可靠性

#### 版本控制
- 按照简化的GitHub使用方案，定期提交代码
- 保持代码的独立性，减少模块间的依赖

## 五、常见问题解答

### 1. 遇到技术问题怎么办？
- 首先尝试通过搜索引擎、AI IDE解决
- 查看相关技术文档和教程
- 在微信群中提问，寻求团队成员的帮助
- 与项目负责人单独沟通

### 2. 如何测试自己的功能？
- 前端：使用微信开发者工具进行预览和调试
- 后端：使用Postman或curl测试API接口
- 整合测试：将前端和后端连接起来，进行完整的功能测试

### 3. 开发进度落后怎么办？
- 及时与项目负责人沟通，说明情况
- 调整开发计划，优先实现核心功能
- 寻求团队成员的协助

## 六、总结

我们的项目采用模块化开发方式，每个成员负责一个完整的功能模块，包括前端页面、后端API和数据库设计。这种方式可以降低沟通成本，让你能够独立完成一个完整的功能，体验全栈开发的流程。

请严格遵循统一开发标准，定期提交进度和代码，遇到问题及时沟通。我们是一个团队，大家互相学习、互相帮助，共同完成这个项目！
